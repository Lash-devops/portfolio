# Интернет-магазин одежды — «ATELIER»

Демо-проект из портфолио [Lash](https://lash-devops.github.io/portfolio/). Витрина магазина одежды с каталогом и работающей корзиной.

**Сложность:** 🔴 Сложный
**Живое демо:** https://lash-devops.github.io/portfolio/projects/clothing-store/

## Что это

Самый сложный из трёх проектов: помимо вёрстки здесь есть управление состоянием — корзина, фильтры и сохранение между визитами. Имитирует работу настоящей витрины интернет-магазина.

## Возможности

- 🧺 **Каталог** из 8 товаров с рендером из JS-массива
- 🔖 **Фильтр по категориям** (верх / низ / обувь / аксессуары)
- 🛒 **Корзина (drawer)** — выезжающая панель: добавление, изменение количества, удаление, подсчёт суммы
- 💾 **Сохранение корзины** в `localStorage` (не теряется при перезагрузке)
- 🔔 **Тост** при добавлении товара
- 🚚 **Прогресс-бар** до бесплатной доставки (порог 10 000 ₽)
- 🧹 Кнопка «Очистить корзину» и демо-оформление заказа
- 📱 Полностью адаптивно (проверено от 320px), сетка 2→4 колонки

## Технологии

- HTML5 (семантическая разметка)
- Tailwind CSS (Play CDN)
- Vanilla JavaScript — состояние корзины, делегирование событий, `localStorage`, рендер из данных
- Шрифты Google Fonts (Playfair Display + Inter)

## Архитектура

Товары описаны массивом `PRODUCTS` в `<script>`. Состояние корзины хранится в объекте `cart` (`{ [id]: количество }`) и синхронизируется с `localStorage`. UI перерисовывается функциями `renderProducts()` и `renderCart()`. Клики обрабатываются делегированием на контейнерах каталога и корзины.

## Чему учит проект

Управление состоянием на чистом JS, рендер интерфейса из данных, делегирование событий, сохранение в `localStorage`, работа с выезжающими панелями и подсчёт значений (сумма, прогресс).

## Запуск

Открой `index.html` в браузере — сборка не нужна.

## Что можно заменить под себя

Товары (название, цена, категория, фото) — в массиве `PRODUCTS`; порог бесплатной доставки — в константе `FREE_SHIPPING`.
